# CS-300 Data Structures and Algorithms

**What was the problem you were solving in the projects for this course?**
The core problem was to design and develop an advising assistance program for the Computer Science department at ABC University (ABCU). The software needed to parse course data and prerequisites from a CSV file, load it into an efficient data structure, print a comprehensive alphanumeric list of all available courses, and provide a fast search mechanism for advisors to look up specific course details and prerequisites.
**How did you approach the problem? Consider why data structures are important to understand.**
I approached the problem by first analyzing and developing pseudocode for three distinct data structures: Vectors, Hash Tables, and Binary Search Trees (BST). Understanding the underlying mechanics of these data structures was critical to selecting the right tool for the job. By evaluating the Big-O time complexity, I determined that while Hash Tables provided rapid $O(1)$ search times, they lacked the inherent order required to efficiently print the alphanumeric course list. Vectors required computationally expensive $O(N \log N)$ sorting. Ultimately, the Binary Search Tree was chosen because an in-order traversal naturally yielded a sorted list in $O(N)$ time, while still offering highly efficient $O(\log N)$ search operations.
**How did you overcome any roadblocks you encountered while going through the activities or project?**
A significant roadblock was managing C++ pointers and memory allocation, specifically when implementing the node removal logic and destructor for the Binary Search Tree, as well as handling collision chains in the Hash Table assignments. I overcame these challenges by breaking down complex pointer manipulations into smaller, recursive helper functions. For example, when removing a BST node with two children, I implemented logic to safely locate the in-order successor and reassign the structural pointers without causing memory leaks or orphaned nodes.
**How has your work on this project expanded your approach to designing software and developing programs?**
This project expanded my approach by demonstrating the immense value of up-front algorithmic planning and runtime analysis. Creating the pseudocode and conducting the runtime analysis before writing the actual C++ implementation shifted my focus from simply making code work to making it highly optimized. It taught me to design software by matching the specific operational needs of the end-user to the inherent strengths and weaknesses of specific data structures.
**How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?**
My development process has evolved to prioritize modularity, encapsulation, and comprehensive documentation. Throughout this project, I structured my code using distinct classes to completely hide the internal tree mechanics and file-parsing complexities from the main application loop. By adhering to single-responsibility principles, enforcing consistent C++ naming conventions, and utilizing explicit inline comments to explain complex recursive algorithms, the resulting software is much easier for future developers to read, maintain, and adapt.
